# 🎪 PartyGen — 语音房 AI 派对游戏生成器

**一句话,生成全房间能玩的派对游戏。** 房主用自然语言描述想玩的游戏("来一个周杰伦歌曲的猜歌游戏"),AI 选择匹配的玩法模板并生成内容,房间内所有人实时同玩。这是核心链路 Demo:自然语言 → 多人可玩的房间游戏。

## 快速开始

```bash
npm install
export ANTHROPIC_API_KEY=sk-...   # 可选;不配置则进入演示模式(内置内容,全部玩法仍可体验)
npm start                          # http://localhost:3000
```

手机/浏览器打开 → 创建房间 → 把 4 位房间号发给朋友(或开多个浏览器标签页模拟多人)→ 描述想玩的游戏 → 生成 → 开始。

```bash
npm test   # 冒烟测试:9 个模板 × (4人随机行为完整局 + 全员挂机超时保护)
```

## 支持的 9 种玩法模板

| 模板 | 玩法 | 交互形态 |
|---|---|---|
| quiz | 抢答问答 | 全员抢答,先答对分高 |
| undercover | 谁是卧底 | 私有发词 + 语音描述 + 投票 |
| draw | 你画我猜 | 实时画板同步 + 打字猜词 |
| bomb | 数字炸弹 | 轮流猜数,范围收缩,踩雷受罚 |
| vote | 投票趣答 | "谁最可能XX"互投上榜 |
| react | 快速反应 | 目标色抢拍,拍错扣分 |
| chain | 词语接龙 | 轮流接词,超时出局 |
| emoji | 表情猜词 | 看 emoji 猜成语/电影/菜名 |
| wheel | 真心话转盘 | 转盘选人语音回答,全员评审 |

AI 生成的是「模板选择 + 内容 spec」(题目、词对、投票问题等),不是任意代码——生成结果先过 schema 校验、失败自动回喂修正(最多 3 次),再交付房间。

## 架构

```
浏览器(渲染层) ←WebSocket→ 房间服务(状态权威)
                                │
                     游戏引擎:确定性状态机
                     模板接口 init / onAction / onTick
                                │
                     生成管线:自然语言 → Claude → {templateId, spec} → 校验/自修复
```

设计原则(为什么这样做):

- **LLM 不写网络代码,只产内容。** 多人同步的复杂度锁在引擎和模板里,由人写一次、所有生成的游戏共享。生成失败面从"代码跑不跑得起来"缩小为"内容 schema 对不对"。
- **服务端状态权威 + 定序广播。** 玩家操作发给服务端,模板的纯函数 `onAction(state, playerId, action, rng)` 计算新状态后带序号广播;客户端只渲染 `state.ui`,天然支持中途加入(快照)和断线重连。
- **确定性随机。** 模板只能用注入的种子化 rng,同一事件序列必产生同一状态。
- **掉线保护内置在引擎节奏里。** 每个模板的 `onTick` 都有超时推进(自动跳过、代猜中间值、按票结算),冒烟测试专门验证"全员挂机也能自然终局"。
- **私有信息通道。** `state.privateUi[playerId]`(卧底词、画家的词)只发给对应玩家,公共 `ui` 广播给全房。

## 目录

```
server/
  engine.js        确定性状态机基础(rng / baseState / rank)
  registry.js      模板注册表 + 给 LLM 的模板说明书
  games/*.js       9 个玩法模板(纯逻辑,无 IO)
  gen/generate.js  生成管线(Claude 调用 + schema 校验 + 自修复回路 + 演示模式回退)
  index.js         房间服务(Express + ws:定序广播、快照、tick 循环、房间回收)
public/            移动优先前端(大厅 / 房间 / 游戏容器 + 各玩法渲染器)
test/smoke.js      多人随机行为冒烟测试
```

## Demo 之后的路线

语音层接 Agora/ZEGO(本 Demo 刻意不含语音,开着微信语音即可体验完整玩法);生成管线加 prompt 缓存与热门游戏库沉淀;客户端套原生壳(WebView 容器)上双端。

## 环境变量

| 变量 | 说明 |
|---|---|
| `ANTHROPIC_API_KEY` / `ANTHROPIC_AUTH_TOKEN` | Claude API key,未设置则演示模式 |
| `ANTHROPIC_BASE_URL` | 自定义 API 网关(可选) |
| `PARTYGEN_MODEL` | 生成模型,默认 `claude-sonnet-5` |
| `PORT` | 服务端口,默认 3000 |
